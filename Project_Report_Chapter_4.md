4.1	PREAMBLE RESULT
This project introduced feature extraction using ICA (Independent Component Analysis) for thumbprints and MediaPipe for nasal landmarks, and the selected features were classified using k-NN (k-Nearest Neighbors). This was implemented on Python using the Django framework. Figure 4.1 expressed the Graphical User Interface built on the Django environment describing the training, registration, and testing process of the multimodal biometric library system. The dataset contains 50 multimodal images (thumbprints and nasal landmarks) collected from 50 subjects within the LAUTECH community. Independent Component Analysis (ICA) and MediaPipe were used for feature extraction and the training and classification were carried out using k-Nearest Neighbors (k-NN). The performance of the Multimodal System (Nasal + Thumbprint) and the Unimodal System (Thumbprint only) were evaluated using metrics such as False Positive Rate (FPR), sensitivity, specificity, accuracy, and computation times.

4.4	DISCUSSION OF RESULTS
The results obtainable in sections 4.2 and 4.3 show significant variation in performance with an increase in threshold value, and the best result is obtained at the threshold value of 0.75 across all metrics (FPR, sensitivity, specificity, and recognition accuracy). The recognition accuracy generated by the Multimodal System (Nasal + Thumbprint) at the optimum threshold value was 98.00% at 0.00% FPR, while that generated by the Unimodal System (Thumbprint only) at the same optimum threshold value was 88.00% at 10.00% FPR respectively. 

The aforementioned results were determined based on the optimum threshold value which happened to be selected because of its outstanding performance compared to other threshold values in securing library access. Therefore, the performance of the developed technique is highly dependent on the threshold value. Figures 4.2, 4.3, 4.4, and 4.5 present the scatterplots of the metrics.

In view of the above results, the Multimodal System (combining nasal geometry with thumbprint ridge patterns) using ICA and k-NN is significantly more accurate, sensitive, and specific at slightly higher but highly efficient computation time, due to a high number of true positives and a very low number of false positives, leading to exceptionally high accuracy and security for the LAUTECH library management system.
